# Rewritegen -> Moldstud old block integration audit

Date: 2026-04-02

## Goal

Prepare a safe article-level integration layer so block-based HTML generated by Rewritegen can render inside the old Moldstud blog without changing the global site theme.

## Moldstud old audit

### Current article render path

- Page entry: `app/(website)/articles/[alias]/page.tsx`
- Server component: `components/articles/article-page.tsx`
- Article HTML insertion point: `<article className={styles['article-body']} dangerouslySetInnerHTML={{__html: article.content}} />`
- Current article styles: `styles/scss/articles/article-page.module.scss`

### Best integration point

- Global SCSS import path: `styles/scss/main.scss`
- Safe scope strategy: apply Rewritegen styles only when the article HTML looks like Rewritegen output.
- Implemented root scope: `rg-article-body`

### Why this scope is required

Rewritegen content uses generic class names such as `.card`, `.head`, `.tag`, `.muted`, `.split`, `.mini`, `.list`, `.btn`. These would conflict with the rest of the site if imported globally. The fix is to scope them under `rg-article-body` only.

### Conflict-prone styles

Potentially dangerous if moved globally:

- `.card`
- `.head`
- `.tag`
- `.muted`
- `.p`
- `.split`
- `.mini`
- `.list`
- `.btn`
- `.foot`
- `.table-wrap`
- `.score`
- `.bar`
- `.num`

Conflict mitigation implemented:

- these selectors are only active under `.rg-article-body`
- `.rg-article-body` is added only when `article.content` matches Rewritegen markers
