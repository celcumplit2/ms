# TASK — FIGMA AUDIT

Задача.
Через Figma и при необходимости через Playwright проведи технический аудит макета, который предполагается внедрить в новую версию сайта Moldstud.

Что нужно получить.
Создай файл docs/figma/FIGMA_AUDIT.md и зафиксируй в нём:
- список страниц/вкладок Figma;
- список ключевых фреймов и экранов;
- какие экраны относятся к desktop / tablet / mobile;
- какие блоки повторяются между экранами;
- где используются компоненты, а где нарисованы уникальные фрагменты;
- какие секции являются базовыми для сайта, а какие декоративными или концептуальными;
- общую структуру экранов: hero, feature sections, grids, cards, forms, navigation, footer, modal patterns и т.д.;
- всё, что выглядит недоделанным, конфликтным или неполным.

Требование.
Не ограничивайся визуальным описанием. Нужна инженерная инвентаризация макета как источника будущей UI-системы.

Результат должен отвечать на вопросы:
- что именно реально есть в Figma;
- что из этого пригодно для production;
- какие части макета являются системными, а какие нет.
