# MASTER PROMPT FOR CODEX

Контекст.
Нужно внедрить существующий Figma-дизайн в новую версию сайта Moldstud. Работать не как “верстальщик по скриншоту”, а как инженер, который переводит макет в production-ready UI-систему. Допускается использование Playwright для входа в Figma, обхода страниц, анализа фреймов, снятия скриншотов и фиксации структуры дизайна.

Главная цель.
Преобразовать Figma-макет в:
1. понятную дизайн-систему;
2. набор переиспользуемых компонентов;
3. layout-правила;
4. план внедрения в новую версию Moldstud;
5. реальную реализацию в коде.

Ограничения.
- Не выдумывай UI вне макета без явной необходимости.
- Не делай pixel-perfect ради pixel-perfect, если это ломает production-логику.
- Не подменяй системную работу одноразовой вёрсткой конкретных экранов.
- Не меняй бизнес-сущности сайта под удобство макета.
- Не скрывай пробелы: всё, чего нет в Figma, фиксируй отдельно.

Принцип исполнения.
Сначала аудит и инвентаризация, потом дизайн-система, потом маппинг, потом внедрение. Не перескакивай сразу к полной сборке страниц.

Что нужно сделать.
1. Исследовать Figma-файл и зафиксировать структуру экранов, секций и компонентов.
2. Выделить дизайн-токены и повторяющиеся паттерны.
3. Сопоставить Figma с реальными сущностями нового Moldstud.
4. Зафиксировать пробелы, конфликты и зоны, где нужна адаптация.
5. Реализовать UI-систему и страницы на основе этого анализа.
6. Подготовить финальный отчёт с проверяемым списком того, что реально сделано.

Формат работы.
- Все промежуточные выводы фиксируй в markdown внутри репозитория.
- Все решения должны быть проверяемыми.
- Если чего-то нет в макете, не молчи: добавь в GAP_REPORT.md.
- В конце дай краткий отчёт по этапам: что сделано, что не сделано, что требует ручного решения.
