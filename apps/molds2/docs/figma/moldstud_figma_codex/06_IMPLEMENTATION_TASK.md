# TASK — IMPLEMENTATION

Задача.
На основании аудита, инвентаризации и дизайн-системы внедри Figma-дизайн в новую версию сайта Moldstud через нормальную компонентную архитектуру.

Приоритет внедрения.
1. base tokens and primitives;
2. shared UI components;
3. shared layout system;
4. ключевые страницы;
5. вторичные страницы и остаточные блоки.

Что нужно сделать.
- внедрить design tokens;
- собрать базовые UI primitives;
- собрать переиспользуемые shared components;
- реализовать layout-правила контейнеров, сеток, секций и внутренних отступов;
- перенести ключевые экраны/блоки из макета;
- адаптировать макет под реальные данные и реальные сущности сайта;
- не допустить одноразочной жёсткой вёрстки, которую нельзя масштабировать.

Обязательный артефакт.
Создай docs/figma/IMPLEMENTATION_PLAN.md, где кратко зафиксируй:
- порядок внедрения;
- какие страницы идут первыми;
- какие блоки являются критическим path;
- какие зависимости есть между задачами.

Требование.
Главный критерий — не похожесть “на картинку”, а получение масштабируемой и поддерживаемой front-end системы, визуально соответствующей Figma и пригодной для развития Moldstud.
