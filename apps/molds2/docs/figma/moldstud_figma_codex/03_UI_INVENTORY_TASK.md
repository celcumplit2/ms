# TASK — UI INVENTORY

Задача.
После аудита макета собери полную инвентаризацию UI-сущностей, которые нужно перенести в код для новой версии Moldstud.

Что нужно получить.
Создай файл docs/figma/UI_INVENTORY.md и разложи по группам:
- navigation;
- buttons;
- inputs / selects / textareas / checkboxes / radios;
- cards;
- badges / chips / tags;
- tables / data blocks;
- tabs / accordions / toggles;
- modals / popovers / tooltips;
- hero blocks;
- content sections;
- pricing / CTA blocks;
- article/blog patterns;
- forms and lead capture blocks;
- footer and utility blocks.

Для каждой сущности зафиксируй:
- где она встречается;
- повторяется ли она;
- является ли она базовым компонентом или шаблоном секции;
- есть ли состояния hover/active/focus/disabled/error;
- есть ли адаптивные вариации.

Требование.
Тебе нужен не просто список красивых элементов, а рабочий каталог того, что должно стать компонентной библиотекой проекта.
