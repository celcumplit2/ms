# TASK — QA / ACCEPTANCE / REPORTING

Задача.
После внедрения проведи финальную самопроверку и зафиксируй результат в отчёте, без попыток выдать недоделанное за завершённое.

Что нужно получить.
Создай финальный docs/figma/REPORT.md.

В отчёте зафиксируй:
- какие этапы выполнены;
- какие документы созданы;
- какие компоненты реально реализованы;
- какие страницы реально реализованы;
- что совпадает с Figma уверенно;
- где есть сознательные отступления;
- что осталось открытым;
- какие места требуют ручного решения или уточнения.

Проверки.
- UI должен быть системным, а не набором разрозненных кусков.
- Компоненты должны быть переиспользуемыми.
- Layout должен быть единым.
- Адаптив не должен быть случайным.
- Никакие пробелы в Figma не должны быть скрыты.
- Все выводы должны быть выражены в PASS/FAIL или clearly pending.

Финальное правило.
Нужен честный инженерный отчёт, а не оптимистичный пересказ.
