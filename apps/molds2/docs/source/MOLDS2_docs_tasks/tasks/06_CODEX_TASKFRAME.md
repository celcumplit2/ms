# MOLDS2 — рамка постановки задач для Codex

## Общий режим работы
Codex должен работать в режиме точного воспроизведения существующего сайта на нашей системе. В этой задаче не нужен продуктовый дизайн, не нужен UX-рефакторинг и не нужна самостоятельная интерпретация.

## Что ему нужно понимать
- задача ограничена первым этапом;
- переносится не весь сайт, а только верхний публичный контур плюс 10 блоговых страниц;
- оригинал является визуальным и структурным эталоном;
- техническая реализация может быть нашей, но внешний результат не должен дрейфовать.

## Что нужно требовать от него по каждому таску
Каждый таск должен быть ориентирован на конкретный результат:
- что исследовать;
- что собрать;
- что воспроизвести;
- что проверить;
- какой артефакт должен появиться.

## Чего от него не нужно
Не нужно просить его:
- предлагать улучшения дизайна;
- менять структуру ради удобства;
- переупаковывать контент в новую редакционную подачу;
- расширять scope без команды.

## Формат полезной постановки
Хорошая задача для Codex в рамках MOLDS2 всегда:
- привязана к конкретному фронту;
- ограничена конкретной областью работ;
- ориентирована на проверяемый результат;
- не оставляет пространства для творческого отступления от оригинала.
