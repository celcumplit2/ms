# MOLDS2 — базовая схема репозитория и деплоя

## Репозиторий
Проект должен создаваться как отдельный Git-репозиторий MOLDS2 с понятной структурой и без мусорных экспериментальных ответвлений.

## Технологическая база
- Node.js;
- структура проекта, пригодная для сборки и сопровождения;
- совместимость с нашим стандартным подходом, как в VIX и Warfair.

## Деплой
Целевой деплой — Cloudflare Pages.

Это означает, что проект должен быть:
- собираемым;
- предсказуемым по output;
- без привязки к ручной магии на локальной машине.

## Базовое ожидание
В конце первого этапа должен существовать проект, который можно:
1. запустить локально;
2. проверить визуально;
3. закоммитить в Git;
4. отправить на Cloudflare Pages.

## Инженерный смысл
Этот baseline нужен не как формальность, а как проверка того, что мы строим не макет и не временный прототип, а реальный переносимый сайт на нашей платформе.
