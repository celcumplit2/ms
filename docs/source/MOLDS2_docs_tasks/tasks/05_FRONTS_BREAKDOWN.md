# MOLDS2 — разбиение на фронты

## Фронт 1. Исследование и съем структуры
Задача фронта:
- зайти на главную страницу;
- через Playwright обойти доступные публичные переходы;
- собрать список страниц первого контура;
- понять, какие типы страниц существуют в зоне первого этапа;
- выделить 10 блоговых страниц для показательного переноса.

Результат фронта:
- карта страниц первого этапа;
- список URL на перенос;
- предварительная карта шаблонов.

## Фронт 2. Визуальное воспроизведение
Задача фронта:
- повторить хедер, футер, основные секции, сетки, карточки, типографику, spacing, CTA, контентные блоки и паттерны внутренних страниц.

Результат фронта:
- визуально идентичная оболочка сайта;
- повторяемые компоненты для ключевых блоков.

## Фронт 3. Шаблоны страниц
Задача фронта:
- собрать отдельные шаблоны для главной;
- собрать шаблоны для страниц первого уровня;
- собрать шаблон для блоговой страницы;
- обеспечить единый подход к повторяющимся секциям.

Результат фронта:
- набор воспроизводимых page templates;
- основа для дальнейшего масштабирования.

## Фронт 4. Контент и маршрутизация
Задача фронта:
- перенести контент без вольной редакторской правки;
- завести маршруты под страницы первого этапа;
- обеспечить корректную связанность переходов.

Результат фронта:
- рабочая локальная версия с полноценной навигацией.

## Фронт 5. Инфраструктура проекта
Задача фронта:
- развернуть репозиторий;
- настроить проект на Node.js;
- привести сборку к виду, совместимому с Cloudflare Pages;
- подготовить стандартный workflow разработки.

Результат фронта:
- проект, который можно собрать, закоммитить и задеплоить.

## Фронт 6. QA на идентичность
Задача фронта:
- проверить соответствие главной;
- проверить соответствие ключевых страниц;
- проверить навигацию;
- проверить, что нет лишних отклонений.

Результат фронта:
- подтверждение того, что получился именно воспроизведенный сайт, а не вариация на тему.
