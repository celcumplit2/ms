# MOLDS2 — манифест проекта

## Суть проекта
MOLDS2 — это не редизайн, не переосмысление и не “улучшенная версия” исходного сайта. Это проект точного воспроизведения существующего публичного сайта на нашей собственной технической базе.

Цель проекта — доказать и практически реализовать способность:
- снять публичную структуру сайта;
- воспроизвести внешний вид и поведение ключевых страниц;
- поднять это на нашем стекe;
- развернуть результат через Git + Cloudflare Pages;
- получить фундамент для дальнейшего масштабирования на большой массив страниц.

## Главный принцип
Снаружи — максимально идентично оригиналу.
Внутри — наша архитектура, наш репозиторий, наш пайплайн, наш деплой.

Нас не интересует творческая интерпретация. Нас интересует воспроизводимость.

## Что считается успехом
Успехом считается не “хорошо сделанный новый сайт”, а сайт, который:
- визуально узнается как оригинальный без оговорок;
- повторяет структуру главной страницы;
- повторяет страницы, доступные с главной через хедер, футер и явную навигацию;
- демонстрирует перенос внутренних шаблонов на примере набора блоговых страниц;
- готов к дальнейшему масштабированию на десятки и сотни тысяч URL.

## Что запрещено
В рамках первого этапа запрещено:
- самовольно менять визуальный стиль;
- улучшать UX по своему усмотрению;
- упрощать структуру, если она существует в оригинале;
- заменять контент новым, если старый доступен публично;
- внедрять новые сущности, которых нет в исходнике;
- спорить с задачей “почему так”, если это не критическая техническая блокировка.

## Что допускается
Допускается:
- использовать наши внутренние компоненты и наш стек, если результат снаружи идентичен;
- применять Playwright, парсинг DOM, ручной аудит и снимки структуры;
- собирать шаблоны страниц на основе повторяющихся блоков;
- нормализовать кодовую базу, если это не меняет внешний результат.

## Философия реализации
1. Сначала снимаем фактуру сайта.
2. Потом строим карту шаблонов.
3. Потом воспроизводим ключевые страницы.
4. Потом закрепляем это как систему, а не как набор случайных копий.

## Практический смысл
Если MOLDS2 собран правильно, мы получаем:
- готовый шаблон переноса больших сайтов;
- отработанную механику клонирования публичной структуры;
- платформенный подход, который потом можно масштабировать на большой объем страниц;
- независимость от исходной реализации оригинального сайта.
